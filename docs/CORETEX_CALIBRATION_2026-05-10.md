### Source of records

`source = challenge-library` against the production challenge package at
`/root/botcoin-coordinator/packages/challenges/dist`. Generated with
`CORETEX_CORPUS_PRODUCTION=1 CORETEX_BIENCODER=pinned CORETEX_LABELER=pinned
CORTEX_REAL_EVAL=1`. That command was valid for this staging record. Current
launch corpus generation refuses any other source or non-pinned bi-encoder and
uses synthesizer-category qrels by default; `CORETEX_LABELER=pinned` is now an
explicit offline audit/A-B path.

### Why this corpus is the calibration corpus, not the full launch corpus

The launch capacity gate (`min-months ≥ 6` at 3 epochs/day) requires
`seeds-per-domain ≈ 512`. This staging run used the older MemReranker-4B
labeler path, so scaling it directly would have made 4B labeling the critical
path. The current launch plan removes that bottleneck by using
synthesizer-category qrels. This calibration corpus uses `seeds-per-domain=4`
(1 752 events) — enough to:

- pass `validate-retrieval-corpus.mjs` with zero errors,
- yield ≥ 4 hard events per family in `eval_hidden` for the bundle's
  hidden pack quotas,
- carry a real `corpusRoot` that the bundle and Phase 13 can pin,
- exercise the full real-model pipeline end-to-end.

The launch corpus is generated by the same corpus generator at
`seeds-per-domain=512`, with production qrels resolved from
`negCategoryRelevanceMap`. The launch run produces a new `corpusRoot` and a
new `bundleHash` that pins the launch corpus root.

`/var/lib/coretex/reports/phase13-real.log`.

## Reproduction

```bash
cd /root/cortex && npm ci && npm run build
node scripts/build-template-bundle.mjs                         # produces /etc/coretex/template-bundle.json
node scripts/download-pinned-models.py /var/lib/coretex/model-cache BAAI/bge-m3
node scripts/download-pinned-models.py /var/lib/coretex/model-cache Qwen/Qwen3-Reranker-0.6B
node scripts/download-pinned-models.py /var/lib/coretex/model-cache IAAR-Shanghai/MemReranker-4B
# (see CORETEX_FINAL_PRODUCTION_E2E_ORCHESTRATOR_RUNBOOK.md Phase 1 for env block)
node scripts/generate-coretex-retrieval-corpus.mjs --source challenge-library \
  --challenge-lib-root /root/botcoin-coordinator/packages/challenges \
  --bundle-manifest /etc/coretex/template-bundle.json \
  --domains companies,quantum_physics,computational_biology,scrna_imputation \
  --seeds-per-domain 4 --modifier-counts 0,1 --constraint-difficulties easy,medium \
  --trap-count 2 --corpus-epoch 0 --out /var/lib/coretex/corpus-epoch-0-calibration.json
bash scripts/orchestrate-cpu-calibration.sh   # validate / determinism × 3 / aggregate / calibrate / build / Phase 13
```

The orchestrator script is idempotent against its own outputs; deleting
`/etc/coretex/bundle-manifest.json` reproduces the same canonical
bundleHash.
